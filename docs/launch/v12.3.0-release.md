# cdxgen v12.3.0

**Release date:** April 2026  
**Previous release:** [`v12.2.1`](https://github.com/cdxgen/cdxgen/releases/tag/v12.2.1)  
**Full changelog:** https://github.com/cdxgen/cdxgen/compare/v12.2.1...v12.3.0

`v12.3.0` is a major release for cdxgen. It expands the project beyond BOM generation with new capabilities for **upstream dependency risk prioritization**, **SPDX conversion/export**, **runtime and container risk analysis**, and broader **ecosystem/source intelligence**.

Since `v12.2.1`, the release range includes **21 commits** spanning new commands, new rule packs, expanded docs, stronger hardening, and better packaging for downstream users.

## 🚀 Highlights

### New: `cdx-audit`
`v12.3.0` introduces **`cdx-audit`**, a new command for **upstream supply-chain risk prioritization** from existing CycloneDX BOMs.

`cdx-audit` can:
- start from one BOM or a directory of BOMs
- resolve supported purls back to upstream source repositories
- generate child SBOMs for upstream projects
- evaluate explainable rules and heuristics
- output results as **console**, **JSON**, or **SARIF**
- support severity-based gating for CI and review workflows

This helps teams answer a practical question quickly:

> Which dependencies should we review first, and why?

### ATT&CK-aware audit and workflow analysis
This release adds stronger security context with:
- **ATT&CK-aware** signals in `cdx-audit`
- expanded GitHub Actions workflow analysis
- richer annotation and triage guidance
- stronger explainability for escalation and manual review

### Better SPDX workflows
SPDX support receives a major upgrade in `v12.3.0`:
- new **`cdx-convert`** command
- improved `cdxgen --format spdx`
- bundled **official SPDX 3.0.1** model and schema assets
- better preservation of advanced CycloneDX semantics during export
- stronger validation and `/sbom` SPDX format support

This improves fidelity for teams that need both **CycloneDX-native generation** and **SPDX-required delivery**.

### New container and runtime risk intelligence
This release expands BOM audit and OBOM analysis with:
- **GTFOBins-backed** container risk analysis
- new **container-risk** audit rules
- expanded **OBOM runtime** rules
- improved Windows **LOLBAS / ATT&CK** triage context
- stronger SOC / IR / compliance guidance

### Broader ecosystem and source intelligence
`v12.3.0` also improves real-world package and source coverage with:
- **PEP 751 `pylock.toml`** support
- purl source resolution for **GitHub**, **Bitbucket**, **Maven**, and **Composer**
- improved **JAR manifest** heuristics for Maven group/name detection
- better handling for **CocoaPods `Podfile.lock`**
- correct npm classification for **`devOptional`** dependencies

### Chrome extension scanning
This release adds support for:
- direct **Chrome extension path scanning**
- extension security property extraction
- dedicated extension BOM audit rules
- fixture-backed test coverage using real extension metadata

### Checksums
Each downloadable binary should be accompanied by a matching checksum file:
- `<asset>.sha256`

Verify hashes before execution.

### Quick verification examples

**Linux**
```bash
ASSET="cdx-audit-linux-amd64"
curl -fsSLO "https://github.com/cdxgen/cdxgen/releases/download/v12.3.0/${ASSET}"
curl -fsSLO "https://github.com/cdxgen/cdxgen/releases/download/v12.3.0/${ASSET}.sha256"
sha256sum -c "${ASSET}.sha256"
chmod +x "${ASSET}"
./"${ASSET}" --help
```

**macOS**
```bash
ASSET="cdx-audit-darwin-arm64"
curl -fsSLO "https://github.com/cdxgen/cdxgen/releases/download/v12.3.0/${ASSET}"
curl -fsSLO "https://github.com/cdxgen/cdxgen/releases/download/v12.3.0/${ASSET}.sha256"
shasum -a 256 -c "${ASSET}.sha256"
chmod +x "${ASSET}"
./"${ASSET}" --help
```
