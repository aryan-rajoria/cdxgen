# Lesson 18 — Detecting AI-Generated Code and Authorship Provenance

This lesson shows how cdxgen detects the use of AI, LLMs, and coding agents to
develop, test, review, and triage a project, records the evidence as namespaced
custom properties, and turns it into audit findings that surface the
**copyright and licensing** implications of AI-authored code.

## Goal

By the end of this lesson you should be able to answer:

1. Which signals does cdxgen use to detect AI/LLM/agent involvement, and how
   confident is each?
2. How do I generate a BOM that records AI authorship provenance?
3. How does `cdx-audit` flag AI-authored code, and why does copyright matter?
4. How do I turn detection off when I do not want it?

## Step 1: Understand the detection channels

cdxgen combines several independent channels because no single one captures all
AI activity. Each detected signal carries a confidence, and the overall score is
combined with a **noisy-OR** model (`1 - Π(1 - cᵢ)`, capped at `0.99`):

| Channel                    | Examples                                                                    | Typical confidence |
| -------------------------- | --------------------------------------------------------------------------- | ------------------ |
| Commit trailers / messages | `Co-authored-by: Claude`, `Generated by Devin`, `codex-cli`                 | 0.80–0.98          |
| Bot / machine authors      | `github-copilot[bot]`, `devin-ai[bot]`, `coderabbitai[bot]`                 | 0.95–1.00          |
| Config / instruction files | `CLAUDE.md`, `AGENTS.md`, `.cursorrules`, `.github/copilot-instructions.md` | 0.55–0.70          |
| CI / review workflows      | `anthropics/claude-code-action`, `coderabbitai`, `openai/codex`             | 0.85–0.90          |
| Branch names               | `copilot/…`, `cursor-…`, `devin/…`                                          | 0.55–0.70          |

Commit-history channels inspect only the most recent commits (default 20) —
recency is enough to prove active AI use and keeps the scan fast.

## Step 2: Generate a provenance BOM

Detection at generation time is opt-in via the `ai-provenance` project type:

```bash
cdxgen -t ai-provenance -o bom.json .
```

The signals are written to the BOM document root `properties` array:

```bash
grep "cdx:ai:codegen" bom.json
```

```text
cdx:ai:codegen:detected = true
cdx:ai:codegen:confidence = 0.99
cdx:ai:codegen:confidence:band = high
cdx:ai:codegen:tools = claude-code,github-copilot
cdx:ai:codegen:phases = dev,ci
cdx:ai:codegen:tool:claude-code = confidence=0.98;phases=dev;channels=config-file;evidence=CLAUDE.md
cdx:ai:codegen:evidence:json = [{"channel":"config-file","tool":"claude-code", ...}]
```

Projects with no AI signals emit **no** `cdx:ai:codegen:*` properties, so ordinary
BOMs stay clean.

## Step 3: Audit for copyright and licensing risk

AI provenance detection is **enabled by default** in `cdx-audit`. Running an audit
turns the properties into findings from the `ai-provenance` rule category:

```bash
cdx-audit --bom bom.json --direct-bom-audit --categories ai-provenance
```

```text
AIPROV-001  AI/LLM/agent involvement detected (claude-code,github-copilot);
            AI-authored code may receive no implicit copyright protection
            without sufficient human authorship.
AIPROV-002  High-confidence AI/LLM/agent authorship detected; review copyright,
            licensing, and human-authorship posture before relying on ownership.
```

If the BOM already carries `cdx:ai:codegen:*` properties they are reused;
otherwise cdx-audit scans the working/target directory and injects them before
evaluating rules.

## Step 4: Why copyright matters

Under current U.S. Copyright Office guidance (_Copyright and Artificial
Intelligence, Part 2_, 2025) and _Thaler v. Perlmutter_ (2025), copyright
requires **human authorship**. Prompts alone do not make the user the author of
an AI system's output. As a result:

- Code produced solely by AI **may receive no implicit copyright protection** and
  could be freely used by others unless protected by a licence, contract, or
  trade secret.
- AI-**assisted** code can still be protected where a human contributes
  sufficiently creative work (substantial modification, or creative selection,
  coordination, and arrangement of AI output).

The takeaway is to **document human creative contributions**. The
`cdx:ai:codegen:*` properties and `AIPROV-*` findings give you an auditable
record to start from. (This is informational, not legal advice, and law varies by
jurisdiction.)

## Step 5: Turn detection off

To disable detection during an audit:

```bash
cdx-audit --bom bom.json --direct-bom-audit --no-ai-provenance
```

At generation time, simply omit `-t ai-provenance` — detection does not run on
ordinary `cdxgen` scans.

model, and property reference.
